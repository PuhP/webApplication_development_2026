# QueueLab — веб-приложение для управления очередями студентов

Клиент-серверное учебное приложение по предмету **«Разработка веб-приложений»**. Проект собран по материалам из LaTeX/PDF-архива: управление очередями студентов на сдачу лабораторных работ и посещений, роли «Студент», «Преподаватель» и «Администратор/староста», авторизация, PostgreSQL, Flask, Jinja2 и Native JS.

## 1. Что реализовано

- Регистрация и вход в систему.
- Ролевая модель:
  - **Студент**: запись в очередь, подтверждение присутствия, выход из очереди, загрузка работ, просмотр своих оценок.
  - **Преподаватель**: создание дисциплин и очередей, вызов следующего студента, перемещение записей, удаление записей, фиксация результата защиты, журнал успеваемости.
  - **Администратор/староста**: управление группами и пользователями, модерация очереди.
- FIFO-алгоритм очереди с возможностью ручного перемещения записей.
- Поля записи в очередь: номер лабораторной, количество работ, предпочтение времени, примерная длительность, комментарий, подтверждение присутствия.
- WebSocket-обновление очереди в реальном времени + резервное обновление через Fetch API.
- PostgreSQL-схема БД в `database/schema.sql`.
- CSV-выгрузка очереди и журнала.
- Адаптивный интерфейс на HTML5, CSS3, Jinja2 и чистом JavaScript.

## 2. Быстрый запуск через Docker

Самый простой способ запустить проект «из коробки» — Docker Compose. Нужны Docker Desktop и Visual Studio Code.

```bash
cd student_queue_app
docker compose up --build
```

После запуска откройте в браузере:

```text
http://localhost:5000
```

При первом старте приложение автоматически создаст таблицы и демо-данные.

### Демо-аккаунты

| Роль | Email | Пароль |
|---|---|---|
| Администратор | `admin@example.com` | `admin1234` |
| Преподаватель | `teacher@example.com` | `teacher1234` |
| Студент | `student@example.com` | `student1234` |

Коды для самостоятельной регистрации ролей:

- преподаватель: `teacher2026`;

## 3. Локальный запуск без Docker

Установите Python 3.11+ и PostgreSQL 14+.

Создайте БД:

```bash
createdb queue_db
```

Создайте виртуальное окружение и установите зависимости:

```bash
cd student_queue_app
python -m venv .venv
```

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

macOS/Linux:

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

Создайте файл `.env` на основе `.env.example` и поменяйте строку подключения:

```text
DATABASE_URL=postgresql+psycopg2://postgres:postgres@localhost:5432/queue_db
```

Запустите приложение:

```bash
python run.py
```

Вариант для быстрой проверки без PostgreSQL:

```text
DATABASE_URL=sqlite:///queue_app.sqlite3
```

PostgreSQL всё равно указан как основной вариант, потому что именно он выбран в теоретической части.

## 7. Основные сценарии проверки

1. Войти как студент `student@example.com / student1234`.
2. Открыть активную очередь и нажать «Подтвердить присутствие» или «Выйти из очереди».
3. Войти как преподаватель `teacher@example.com / teacher1234` в другом браузере.
4. Открыть ту же очередь, нажать «Вызвать следующего», переместить запись или отметить лабораторную как сданную.
5. В студенском окне очередь обновится через WebSocket.
6. Открыть «Журнал» и скачать CSV.
7. Войти как администратор и проверить создание групп/пользователей.

## 8. Схема БД

Ключевые таблицы:

- `users` — пользователи и роли;
- `study_groups` — учебные группы;
- `subjects` — дисциплины;
- `queue_sessions` — конкретные очереди на занятия;
- `queue_entries` — записи студентов в очередь;
- `grade_records` — журнал успеваемости;
- `attendance` — посещаемость и подтверждение присутствия;
- `work_uploads` — загруженные работы студентов.

Полная схема находится в `database/schema.sql`.

## 9. Замечания по учебной эксплуатации

Проект предназначен для локального запуска и демонстрации курсовой работы. Для промышленного размещения нужно дополнительно настроить HTTPS, production WSGI/ASGI-сервер, резервное копирование БД, CSRF-защиту форм и отдельное хранилище файлов.
