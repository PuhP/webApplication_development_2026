# Что реализовано по материалам из `Documens.rar`

Из LaTeX-файлов и архитектурной схемы были выделены следующие требования:

- стек: Python, Flask, SQLAlchemy, PostgreSQL, Jinja2, HTML5/CSS3, Native JS;
- роли: студент, преподаватель, администратор/староста;
- регистрация, авторизация и разграничение доступа по ролям;
- FIFO-очередь студентов на сдачу лабораторных работ;
- данные записи: номер лабораторной, количество работ, предпочтение по времени, примерная длительность, подтверждение присутствия;
- преподаватель и администратор могут вызывать студента, перемещать записи, удалять записи из очереди;
- преподаватель фиксирует результат защиты, после чего студент удаляется из активной очереди, а запись появляется в журнале;
- администратор управляет группами и пользователями;
- студент загружает работы и видит свои оценки;
- есть real-time обновление через WebSocket и резервное обновление через Fetch API;
- предусмотрены CSV-отчеты по очереди и журналу.

Файлы с исходными схемами из архива сохранены в папке `docs/original_diagrams/`.
